Assignment 3: Thought Behind Microservices
==========================================

Monolithic Challenges
---------------------

1. Deployment Issues

Entire application redeployed for small changes.

2. Scaling Issues

Whole application scaled instead of a single module.

3. Team Collaboration Issues

Multiple teams work on same codebase.

4. Technology Constraints

Single technology stack.

5. Release Management Issues

One module failure affects entire application.

Why Monolith Becomes Difficult?
-------------------------------

Large codebase
Slow deployment
Difficult maintenance
Poor scalability

How Microservices Solve Problems?
---------------------------------

Independent deployment
Independent scaling
Technology flexibility
Fault isolation

Independent Services
--------------------

User Service
Product Service
Cart Service
Order Service
Payment Service

| Feature              | Monolith    | Microservices      |

Assignment 6: Important Attributes of Microservices
===================================================

1. Independent Deployment

Each service deployed separately.

Example:
Order Service updated without affecting Payment Service.

2. Loose Coupling

Services communicate through APIs.

3. High Cohesion

Each service performs one business function.

4. Scalability

Scale only required service.

5. Fault Isolation

Failure in Payment Service won't crash User Service.

6. Decentralized Data Management

Separate database per service.

7. Technology Heterogeneity

Different services use different technologies.

8. Automation

CI/CD pipelines automate deployment.


Food Delivery Example
---------------------

| Service            | Attribute Implementation |
| ------------------ | ------------------------ |
| User Service       | Independent deployment   |
| Restaurant Service | Separate database        |
| Order Service      | High cohesion            |
| Payment Service    | Fault isolation          |
| Delivery Service   | Independent scaling      |

Assignment 10: Microservices Design Patterns
============================================

| Pattern              | Purpose                 | Problem Solved     | Example              |
| -------------------- | ----------------------- | ------------------ | -------------------- |
| API Gateway          | Single Entry Point      | Multiple Endpoints | Netflix              |
| Service Discovery    | Locate Services         | Dynamic Services   | Eureka               |
| Circuit Breaker      | Failure Handling        | Cascading Failures | Resilience4j         |
| Saga                 | Distributed Transaction | Data Consistency   | Order Processing     |
| CQRS                 | Separate Read/Write     | Performance        | Banking              |
| Event Sourcing       | Store Events            | Audit Trail        | Financial Systems    |
| Database per Service | Data Isolation          | Tight Coupling     | E-Commerce           |
| Strangler            | Gradual Migration       | Monolith Migration | Legacy Modernization |
