Section 1: Conceptual Questions
===============================

1. What is Netflix OSS?
-----------------------

Netflix OSS (Open Source Software) is a collection of tools developed by Netflix to build scalable and resilient microservices applications.

Role in Microservices
Service Discovery
Load Balancing
API Gateway
Fault Tolerance
Distributed Configuration

Key Components
--------------

| Component | Purpose                         |
| --------- | ------------------------------- |
| Eureka    | Service Discovery Server        |
| Ribbon    | Client-side Load Balancer       |
| Zuul      | API Gateway                     |
| Hystrix   | Circuit Breaker/Fault Tolerance |


Brief Description

Eureka

Registers services
Helps services discover each other

Ribbon

Distributes requests among service instances

Zuul

Single entry point for APIs
Routing and filtering

Hystrix

Prevents cascading failures
Provides fallback mechanisms



2. What is Spring Cloud?
------------------------

Definition

Spring Cloud is a framework that provides tools for building distributed and cloud-native microservices applications.

How It Simplifies Microservices

Provides built-in support for:

Service Discovery
Load Balancing
API Gateway
Distributed Configuration
Circuit Breakers
Distributed Tracing

Spring Boot vs Spring Cloud
---------------------------

| Feature                  | Spring Boot        | Spring Cloud              |
| ------------------------ | ------------------ | ------------------------- |
| Purpose                  | Build applications | Build distributed systems |
| Focus                    | Single application | Multiple microservices    |
| Service Discovery        | No                 | Yes                       |
| Load Balancing           | No                 | Yes                       |
| API Gateway              | No                 | Yes                       |
| Configuration Management | Basic              | Centralized               |


3. Service Discovery
--------------------

What is Service Discovery?

A mechanism through which microservices locate and communicate with each other dynamically.

Why Required?

In cloud environments:

IP addresses change
Containers restart frequently
Services scale dynamically

Hardcoding URLs becomes impossible.

What Problem Does Eureka Solve?

Eureka maintains a registry of available services.

Instead of:

http://localhost:8081/employees

Use:

http://EMPLOYEE-SERVICE/employees

Eureka resolves the actual instance.



4. Load Balancing
-----------------

Client-Side Load Balancing

The client chooses which service instance should receive the request.

Example:
--------

Department Service
        |
   Eureka Lookup
        |
Employee Instance 1
Employee Instance 2
Employee Instance 3

Client-side vs Server-side
--------------------------

| Feature        | Client-Side             | Server-Side     |
| -------------- | ----------------------- | --------------- |
| Decision Maker | Client                  | Load Balancer   |
| Example        | Ribbon, Spring Cloud LB | Nginx           |
| Network Hop    | Less                    | Extra Hop       |
| Performance    | Faster                  | Slightly Slower |


Why Ribbon Replaced?

Ribbon is now maintenance mode.

Spring Cloud LoadBalancer:

Lightweight
Better Spring Integration
Actively maintained



5. OpenFeign
------------

What is OpenFeign?

A declarative REST client used for communication between microservices.

Why Declarative?

Instead of writing HTTP calls manually:

@FeignClient(name="employee-service")
public interface EmployeeClient {

    @GetMapping("/employees/{id}")
    Employee getEmployeeById(@PathVariable Long id);

}


Advantages Over RestTemplate
----------------------------

| RestTemplate          | OpenFeign          |
| --------------------- | ------------------ |
| More code             | Less code          |
| Manual URL handling   | Service name based |
| Manual serialization  | Automatic          |
| Difficult maintenance | Easy maintenance   |

Task 3: Department Service (Consumer)
-------------------------------------

RestTemplate Bean
-----------------

@Bean
public RestTemplate restTemplate() {
    return new RestTemplate();
}


Controller
----------

@RestController
@RequestMapping("/departments")
public class DepartmentController {

    @Autowired
    private RestTemplate restTemplate;

    @GetMapping("/{id}/with-employees")
    public String getDepartmentEmployees(
            @PathVariable Long id) {

        return restTemplate.getForObject(
            "http://localhost:8081/employees",
            String.class);
    }
}


Section 3: Spring Cloud LoadBalancer
====================================

Configuration
-------------

@Bean
@LoadBalanced
public RestTemplate restTemplate() {
    return new RestTemplate();
}

Service Call
------------

restTemplate.getForObject(
        "http://EMPLOYEE-SERVICE/employees",
        String.class);

Multiple Instances
------------------
Employee Service 1 → 8081
Employee Service 2 → 8082
Employee Service 3 → 8083


Questions
---------

1. How Does LoadBalancer Choose Instance?

Default algorithm:

Round Robin

Example:

Request1 → 8081
Request2 → 8082
Request3 → 8083
Request4 → 8081


2. What Happens When One Instance Goes Down?

Eureka removes unhealthy instance.

Requests are automatically routed to active instances.





Section 4: OpenFeign Implementation
===================================

Dependency
----------

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>

Enable Feign
------------

@SpringBootApplication
@EnableFeignClients
public class DepartmentApplication {
}

Feign Client
------------

@FeignClient(name="employee-service")
public interface EmployeeClient {

    @GetMapping("/employees/{id}")
    String getEmployeeById(
            @PathVariable Long id);
}

Controller
----------

@RestController
public class DepartmentController {

    @Autowired
    private EmployeeClient employeeClient;

    @GetMapping("/employee/{id}")
    public String getEmployee(
            @PathVariable Long id) {

        return employeeClient.getEmployeeById(id);
    }
}

Order Service Calling Payment Service
-------------------------------------

RestTemplate Bean
-----------------

@Bean
public RestTemplate restTemplate() {
    return new RestTemplate();
}


Service Class
-------------

@Service
public class OrderService {

    @Autowired
    private RestTemplate restTemplate;

    public String getPayment(String orderId) {

        return restTemplate.getForObject(
                "http://localhost:8081/payment/"
                        + orderId,
                String.class);
    }
}
